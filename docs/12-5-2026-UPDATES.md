# 12-5-2026 Updates

- Added a small CLI argument parser so global flags are handled before prompt text, while prompt text like `explain --help` is no longer mistaken for the help command.
- Added one-shot flags: `--provider <id>`, `--model <id>`, `--approval-mode <approval|auto-edits|yolo>`, and `--no-history`.
- Changed one-shot mode to stream assistant text deltas to stdout as they arrive instead of waiting for the full agent loop to finish.
- Lazy-loaded the OpenTUI runtime so TUI dependencies are only imported when launching the interactive UI.
- Persisted one-shot transcripts to the existing history store by default, with `--no-history` as an opt-out.
- Changed CLI error handling so usage errors exit `64`, tool/config problems use distinct non-zero codes where detectable, provider response errors exit `70`, and aborted runs exit `130`.
- Added `recode doctor` to report config, workspace, active provider, selected model, API key presence, approval mode, history writability, and model-listing status.
